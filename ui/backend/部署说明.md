# 部署说明

## 一、树莓派部署（当前使用 mpg123）

### 1. 文件夹结构

在树莓派上，项目应该放在：
```
/home/xiaohehe/A_theme_one/ui/
├── backend/
│   ├── server.py          # 后端服务器
│   └── music/             # 音乐文件夹
│       ├── alarm.mp3      # 闹钟音乐
│       └── white.mp3      # 白噪音音乐
└── [前端文件...]
```

### 2. 安装依赖

#### 安装 mpg123（树莓派老系统推荐）
```bash
sudo apt update
sudo apt install mpg123 -y
```

#### 安装 Python 依赖
```bash
cd /home/xiaohehe/A_theme_one/ui/backend
pip install fastapi uvicorn
# 或使用虚拟环境（推荐）
python3 -m venv venv
source venv/bin/activate
pip install fastapi uvicorn
```

### 3. 启动后端服务

```bash
cd /home/xiaohehe/A_theme_one/ui/backend
python3 server.py
# 或使用 uvicorn（推荐）
uvicorn server:app --host 0.0.0.0 --port 8000
```

### 4. 验证

启动后，你应该看到类似输出：
```
============================================================
SERVER START - Environment Detection
============================================================
Platform: Linux ...
AUDIO_PLAYER: /usr/bin/mpg123
PLAYER_TYPE: mpg123
...
============================================================
```

## 四、需要部署的文件

### 必须部署的文件：
1. **backend/server.py** - 后端服务器代码
2. **backend/music/** - 音乐文件夹（包含所有 .mp3 文件）

### 前端文件（如果需要在前端部署）：
- `api.ts` - API 配置（需要修改 IP 地址）
- `components/` - 所有组件
- `App.tsx`, `index.tsx` - 主文件
- `package.json` - 前端依赖
- 其他前端文件

### 不需要部署的文件：
- `node_modules/` - 可以通过 `npm install` 重新生成
- `.git/` - Git 仓库文件（如果使用 Git）
- `__pycache__/` - Python 缓存文件

## 七、常见问题

### Q: 音乐播放失败？
A: 检查：
1. mpg123 是否已安装：`which mpg123`
2. 音乐文件是否存在：`ls -la /home/xiaohehe/A_theme_one/ui/backend/music/`
3. 文件权限是否正确：`chmod 644 *.mp3`

### Q: 服务无法启动？
A: 检查：
1. Python 版本：`python3 --version`（需要 Python 3.7+）
2. 依赖是否安装：`pip list | grep fastapi`
3. 端口是否被占用：`lsof -i:8000`

### Q: 如何后台运行？
A: 使用 `nohup` 或 `systemd`：
```bash
# 方式1: nohup
nohup uvicorn server:app --host 0.0.0.0 --port 8000 > server.log 2>&1 &

# 方式2: systemd（创建服务文件）
sudo nano /etc/systemd/system/sleep-system.service
```
